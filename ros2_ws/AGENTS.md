# ROS2 WORKSPACE NOTES

## OVERVIEW
`ros2_ws/` is the active ROS 2 workspace. Edit `src/`; treat `build/`, `install/`, and `log/` as disposable outputs.

## STRUCTURE
```text
ros2_ws/
├── src/      # authored packages
├── build/    # generated by colcon
├── install/  # generated install tree
└── log/      # build/test logs
```

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Package source | `src/` | Only authored code worth editing |
| Generated message/build issues | `build/` | Inspect only; regenerate instead of patching |
| Launch/runtime install paths | `install/` | Useful for debugging packaging/layout |
| Recent build/test failures | `log/` | Evidence for colcon runs |

## CONVENTIONS
- Build from this directory with `colcon build`.
- After building, source `install/setup.bash` before `ros2 launch`, `ros2 topic`, or `ros2 service` commands.
- Package tests live beside packages, usually under `src/<pkg>/test/`.
- Keep launch/config files packaged via each package `setup.py` or `CMakeLists.txt`.

## ANTI-PATTERNS
- Do not hand-edit files under `build/` or `install/` to “fix” runtime behavior.
- Do not debug only against installed copies; verify the corresponding file under `src/`.
- Do not add first-party logic directly under `ros2_ws/`; put it in a package.

## COMMANDS
```bash
cd ros2_ws
colcon build
colcon test
colcon test-result --verbose
source /opt/ros/*/setup.bash && source install/setup.bash
```
